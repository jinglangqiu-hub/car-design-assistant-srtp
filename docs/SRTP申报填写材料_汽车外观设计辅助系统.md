# SRTP 项目申报/结项填写材料

项目名称：汽车外观设计辅助系统  
题目建议：基于 StyleGAN2-ADA 的汽车外观设计辅助系统设计与实现  

## 一、项目研究的目的、意义

本项目面向汽车外观设计早期概念生成与方案辅助需求，研究如何利用生成式人工智能技术构建一个能够根据车型选择生成汽车外观概念图的辅助系统。汽车外观设计在早期阶段通常需要大量视觉参考和造型草案，用于启发设计方向、比较车身比例和讨论风格方案。传统人工草图方式具有较强创造性，但生成效率受设计经验和时间限制；普通图片检索又主要依赖已有车型，难以快速形成新的概念组合。因此，探索利用生成模型辅助汽车外观概念设计，具有一定的研究价值和应用意义。

本项目选择 StyleGAN2-ADA 作为核心生成模型，以 CompCars 汽车图像数据为基础，完成汽车外观数据清洗、三类车型数据集构建、云端模型训练、桌面程序开发和安装包制作。项目最终实现了跑车、轿车、SUV 三类汽车外观图像的可选生成，并形成可运行的 Windows 桌面应用程序。该研究不仅训练了生成模型，还将模型能力转化为可交互软件工具，有助于理解人工智能模型从实验训练到实际系统落地的完整过程。

项目的意义主要体现在三个方面：第一，在设计辅助层面，系统能够快速生成多张汽车外观概念图，为早期造型探索提供参考；第二，在技术实践层面，项目完整覆盖数据预处理、深度学习训练、云端迁移、模型部署和桌面端打包流程；第三，在创新训练层面，项目结合 SRTP 结项要求，采用三类车型独立微调模型实现可控生成，为后续开展更高分辨率生成、条件生成和多模态设计控制提供基础。

## 二、项目创新点（或特色）

1. 面向汽车外观设计场景进行生成模型应用，而不是仅停留在通用图像生成实验。项目将生成式模型与汽车外观概念设计需求结合，围绕跑车、轿车、SUV 三类车型建立可选生成系统。

2. 采用“三类数据集 + 三类微调模型”的工程路线实现类型可控生成。相比重新训练复杂的条件生成模型，该方案充分利用已有通用 baseline 权重，在有限时间和算力条件下稳定实现三类车型选择生成。

3. 完成从数据到应用的完整工程闭环。项目不仅包括数据清洗和模型训练，还进一步开发桌面程序，实现车型选择、随机种子控制、truncation 参数调节、单张/批量生成、图片保存和参数记录等功能。

4. 完成云端训练迁移和长时间训练管理。针对本地电脑内存不足、无法长时间开机的问题，项目迁移至 RTX 5090 云端 GPU 环境，并完成 PyTorch、CUDA、模型代码兼容性验证和正式训练。

5. 完成可交付安装包与云端归档。项目使用 PyInstaller 和 Inno Setup 打包为 Win11 可安装桌面应用，并将源码、模型、训练数据集和分卷安装包整理上传至 GitHub 仓库和 Release，便于后续继续开发和复现。

## 三、项目研究的主要内容

本项目研究内容主要包括以下几个方面。

第一，汽车外观数据集构建。项目以 CompCars 汽车图像数据为基础，筛选适合汽车外观生成任务的图像，完成裁剪、清洗、方形化和 256×256 标准化处理，形成 StyleGAN2-ADA 可训练的数据包。随后根据车型属性信息，将清洗后的汽车图像划分为跑车、轿车、SUV 三类数据集，其中跑车 1519 张、轿车 5492 张、SUV 9889 张。

第二，生成模型训练与微调。项目首先基于清洗后的通用汽车外观数据训练 StyleGAN2-ADA baseline 模型，使模型学习汽车外观图像的基本分布；随后以通用 baseline 为初始化权重，分别使用三类车型数据集训练 sports.pkl、sedan.pkl、suv.pkl 三个微调模型，实现三类车型外观的定向生成。

第三，云端训练环境搭建。项目租用 RTX 5090 32GB 显存云端容器实例，使用 Ubuntu 22.04、Python 3.12.3、PyTorch 2.7.0+cu128 和 CUDA Runtime 12.8 环境，完成 GPU 可用性、CUDA 架构、矩阵运算和 StyleGAN2-ADA 训练脚本验证。正式训练采用 nohup 后台运行，并通过日志、PID 文件和 nvidia-smi 进行监控。

第四，桌面程序开发。项目基于 Python Tkinter/ttk 开发汽车外观设计辅助系统桌面程序，集成三类训练模型，支持用户选择跑车、轿车或 SUV，设置随机种子和 truncation 参数，生成单张或多张汽车外观图，并自动保存生成结果和参数记录。

第五，系统打包与成果归档。项目使用 PyInstaller 将 Python 程序、PyTorch 依赖、StyleGAN2-ADA 代码和模型文件打包为桌面程序目录，再使用 Inno Setup 制作安装包。由于安装包体积较大，进一步采用分卷方式便于 GitHub Release 上传。最终形成源码仓库、Release 资产、结项报告、答辩 PPT 和 PPT 素材包等成果。

## 四、项目研究过程与方法

项目研究过程按照“数据准备—模型训练—系统开发—打包部署—成果整理”的路线推进。

在数据准备阶段，项目首先对 CompCars 汽车图像进行整理和清洗。为了让生成模型更稳定地学习汽车外观形态，项目保留主体较完整、外观轮廓较清晰、适合训练的图像，并统一处理为 256×256 方形图像。随后根据车型属性文件编写数据集划分脚本，将原始车型类型映射到跑车、轿车、SUV 三类目标类别，生成三个 StyleGAN2-ADA zip 格式训练数据集。

在模型训练阶段，项目选择 StyleGAN2-ADA 作为生成模型。该模型适合在有限数据集上训练，并可通过自适应数据增强缓解判别器过拟合。项目先完成通用 baseline 训练，再以 baseline 权重作为初始化，分别训练三类车型模型。第一轮训练完成后，三类模型已能生成具有车辆轮廓和类别差异的图像，但质量仍未完全达到展示要求，因此继续进行追加训练。追加训练从第一轮最终模型继续，跑车追加 1000 kimg，轿车追加 1500 kimg，SUV 追加 1500 kimg，最终每类均获得 10 张以上可展示图片。

在云端迁移阶段，项目将训练任务从本地迁移到云端 GPU 实例，解决本地电脑内存有限、无法长时间开机的问题。云端环境经过 PyTorch CUDA 可用性、RTX 5090 架构支持和 StyleGAN2-ADA 冒烟训练验证后，再启动正式训练。训练过程中使用日志文件记录训练进度，并定期检查 GPU 占用、训练 tick、最终模型输出和样例图生成情况。

在系统开发阶段，项目将训练得到的三类模型集成到桌面程序中。程序启动后初始化模型后端，根据用户选择加载对应的 pkl 模型，并使用 seed 和 truncation 参数生成图像。界面经过优化后包含标题区、车型选择区、参数控制区、生成按钮、图片预览区和状态提示区，满足结项演示和普通用户操作需求。

在打包部署阶段，项目解决了 PyInstaller 打包中出现的依赖缺失、StyleGAN2-ADA 代码路径、模型文件复制和打包后运行路径等问题，最终成功生成可运行的桌面程序。随后使用 Inno Setup 制作安装包，并验证安装后的应用能够正常生成汽车图片。最后将代码和文档上传 GitHub，将模型、安装包分卷和三类训练数据集上传 GitHub Release，用于云端归档和后续继续开发。

## 五、项目完成预期目标情况

项目已基本完成立项时设定的主要目标。

首先，项目完成了汽车外观生成模型训练。通过 StyleGAN2-ADA baseline 和三类微调训练，最终得到跑车、轿车、SUV 三个生成模型，分别保存为 sports.pkl、sedan.pkl、suv.pkl。三类模型生成结果之间已经具有明显差异，能够体现跑车较低车身和运动姿态、轿车较均衡比例、SUV 较高车身和厚重体量等基本特征。

其次，项目实现了根据选择生成三类汽车外观图的核心功能。桌面程序中设置了跑车、轿车、SUV 三个选项，用户选择不同车型时，程序会加载对应模型并生成相应类别的汽车外观图。该功能直接对应立项要求中的“三种类型汽车外观图生成”目标。

再次，项目完成了可运行桌面程序开发。系统支持随机种子设置、truncation 生成稳定度调节、单张生成、批量生成、打开输出目录、另存当前图片和参数记录等功能。程序已经在本地运行和 PyInstaller 打包环境中验证成功。

最后，项目完成了安装包制作和成果归档。安装包使用 Inno Setup 分卷生成，安装后应用能够正常启动并生成图片。项目源码已上传 GitHub，模型、三类训练数据集、安装包分卷和样例图已作为 Release 资产上传。项目还完成了结项报告、答辩 PPT、PPT 素材包和相关文档整理。

总体来看，项目达到了“可运行系统、三类车型生成、样例图展示、结项论文和答辩材料”的预期目标。

## 六、项目成果简介

本项目最终形成了较完整的 SRTP 结项成果体系，主要包括以下内容。

1. 三类车型生成模型。项目训练得到 sports.pkl、sedan.pkl、suv.pkl 三个 StyleGAN2-ADA 微调模型，分别用于跑车、轿车和 SUV 外观图生成。

2. 三类训练数据集。项目构建了 compcars_sports_256_square.zip、compcars_sedan_256_square.zip、compcars_suv_256_square.zip 三个实际训练数据集，并保留 srtp_type_mapping.json 类别映射文件。

3. 桌面应用程序。项目开发了汽车外观设计辅助系统桌面程序，支持车型选择、参数输入、图片生成、保存和记录，能够作为结项演示中的核心系统成果。

4. Windows 安装包。项目使用 PyInstaller 和 Inno Setup 完成程序打包，生成 Win11 可安装应用，并采用分卷方式解决安装包体积较大的上传和分发问题。

5. 生成图片样例。项目通过最终模型和桌面程序生成了汽车外观图片样例，每类均能够筛选出 10 张以上可展示图，用于论文、PPT 和系统演示。

6. 项目文档与云端归档。项目整理了 README、云端环境配置记录、训练脚本、结项报告、答辩 PPT、PPT 素材包、GitHub 上传指南等材料，并将源码和 Release 资产上传 GitHub，便于后续继续开发。

## 七、项目研究工作的分析与总结

本项目最大的特点是完成了一个从模型实验到实际桌面应用的完整闭环。相比只训练一个生成模型并展示若干图片，本项目进一步解决了数据分类、云端训练、模型选择加载、桌面交互、打包部署和云端归档等工程问题，使生成模型真正成为一个可运行的辅助系统。

从研究方法看，项目采用三类独立微调模型实现可控生成，是在时间、算力和结项要求之间较为合理的折中方案。单一条件生成模型虽然在理论上更统一，但需要重新整理标签格式、修改训练流程并承担更高调试风险；三类微调模型能够复用通用 baseline，并且在桌面程序中通过模型切换直接实现类别选择，工程稳定性更高。

从训练过程看，云端迁移是项目顺利完成的重要条件。RTX 5090 云端环境解决了本地设备显存和长时间运行限制，使三类模型的正式训练和追加训练能够连续完成。训练后期根据生成质量不足的问题及时追加训练，也体现了项目从结果评估反向调整训练策略的过程。

从系统实现看，桌面程序将深度学习模型封装为可操作界面，降低了使用门槛。系统不仅能生成图片，还能自动保存结果和记录参数，为后续复现实验、筛选样例和展示成果提供便利。打包过程中遇到的依赖缺失、模型路径和安装包体积问题，也推动项目形成了较完整的工程解决方案。

总体而言，本项目基本实现了 SRTP 立项目标，形成了数据、模型、程序、安装包、报告和答辩材料等多层次成果。项目证明了 StyleGAN2-ADA 在汽车外观概念图生成中的可行性，也为后续开展更高分辨率、更强可控性的汽车设计辅助系统提供了基础。

## 八、项目研究存在的问题、建议

项目仍存在以下不足。

第一，当前生成图像分辨率为 256×256，适合结项展示和概念参考，但细节表现有限，无法达到真实汽车设计渲染图或工程设计图的质量。后续可构建 512×512 或更高分辨率数据集，并在更充足算力条件下训练高分辨率模型。

第二，跑车类别样本数量相对较少，仅 1519 张，导致跑车模型在部分随机种子下仍可能出现车身形变、细节模糊或局部结构不稳定。后续可继续扩充跑车类数据，或使用数据增强和迁移学习策略改善少样本类别效果。

第三，当前系统采用三个独立模型实现车型选择，虽然工程稳定，但随着类别数量增加，模型文件数量和维护成本会增加。后续可尝试训练单一条件生成模型，将车型类别作为条件输入，实现更统一、更可扩展的生成架构。

第四，当前交互控制主要包括车型、seed 和 truncation 参数，控制维度仍较粗。后续可引入文本提示、草图输入、车身姿态控制、颜色控制或局部编辑功能，使系统更接近真实设计辅助工具。

第五，当前评价方式以人工筛选和展示性评价为主，缺少系统性的客观指标评估。后续可引入 FID、类别识别准确率、人工问卷评分等方法，从图像质量、类别一致性和设计可用性等角度进行更全面评估。

针对后续研究，建议在现有项目基础上继续保留三类数据集和模型作为基础版本，优先开展高分辨率训练和条件生成模型探索；同时进一步完善桌面程序的图库管理、批量筛选、收藏评分、导出报告等功能，使系统从结项演示原型逐步发展为更完整的汽车外观设计辅助工具。
