# SRTP 项目 TODO.md

## 项目名称

**基于条件生成对抗网络（GAN）的汽车外观设计辅助系统**

## 当前阶段定位

中期报告已经提交，项目从“论证阶段”正式进入“云端训练 + 闭环交付”阶段。  
后续开发不再以本地电脑为主，所有模型训练、数据处理、数据集构建、推理与 Demo 相关工作统一迁移到云服务器完成。本地电脑只用于报告撰写、PPT 制作、结果查看与材料整理。

---

# 一、总目标

在结项前完成一套完整、可信、可展示、可答辩的 SRTP 成果包，包括：

- [ ] 云端训练环境与工作流
- [ ] 正式 18 标签汽车图像数据集
- [ ] 条件生成模型训练结果
- [ ] 基础评估与结果分析
- [ ] 可运行的汽车外观设计辅助 Demo
- [ ] 结项报告 / 论文
- [ ] 答辩 PPT
- [ ] 演示视频
- [ ] 项目代码、模型权重与说明文档

## 项目推进原则

- **结果优先于完美**
- **闭环优先于扩展**
- **系统优先于空谈**
- **可展示优先于炫技**
- **先完成 256 主线成果，再尝试 512 增强实验**
- **先小闭环验证，再扩大数据和训练规模**
- **不纯手工逐张从零打标签，采用规则筛选 + 半自动预标注 + 人工复核**

# 五、Phase 3：正式训练前的数据处理

## 3.1 锁定标签体系

### 车型标签 type

- [ ] sedan
- [ ] SUV
- [ ] sports car

### 颜色标签 color

- [ ] black
- [ ] white
- [ ] red
- [ ] blue
- [ ] silver_gray
- [ ] yellow

### 组合标签

共 18 类：

```text
type × color = 3 × 6 = 18
```

- [ ] 建立 `docs/label_mapping.md`
- [ ] 为 18 个组合标签分配 `label_id`
- [ ] 标签命名固定后不再随意修改

## 3.2 生成候选可用样本集

从 CompCars 原始数据中筛出适合训练的候选图片。

筛除：

- [ ] 非整车图
- [ ] 视角过乱
- [ ] 主体不完整
- [ ] 遮挡严重
- [ ] 分辨率过低
- [ ] 模糊严重
- [ ] 重复图
- [ ] 背景干扰过重
- [ ] 极端透视角度

保留目标：

- [ ] 3/4 前侧视角优先
- [ ] 主体完整
- [ ] 车辆清晰
- [ ] 背景相对可接受
- [ ] 颜色可判断

输出：

- [ ] `data/cleaned/candidate_images/`
- [ ] `docs/dataset_filtering_log.md`

## 3.3 车型标签半自动生成

原则：**不人工逐张从零判断车型，优先利用 CompCars 元信息。**

- [ ] 查看 CompCars 原始目录结构
- [ ] 查看可利用的品牌、车系、车型、年款信息
- [ ] 建立车型到 type 的映射表
- [ ] 编写脚本批量生成 `type_auto`
- [ ] 抽样检查映射是否合理
- [ ] 对错误或模糊类别进行人工修正

输出：

- [ ] `data/labels/type_mapping.csv`
- [ ] `data/labels/type_auto_labels.csv`

## 3.4 颜色标签半自动 / 人工快审

原则：**颜色不建议完全依赖自动算法，也不建议复杂化。**

推荐做法：

- [ ] 先用缩略图进行人工快速分类
- [ ] 必要时用脚本估计主色作为辅助
- [ ] 只分 6 类颜色，不做过细颜色系统
- [ ] 对难以判断、偏色、背景干扰严重的图单独处理
- [ ] 颜色过于模糊的样本可以删除，不强行标注

输出：

- [ ] `data/labels/color_labels.csv`

## 3.5 人工复核

人工只做复核和难例处理，不做全量从零打标签。

- [ ] 复核 type 明显错误样本
- [ ] 复核 color 明显错误样本
- [ ] 删除标签不可靠样本
- [ ] 删除图像质量不合格样本
- [ ] 保证标准前后一致

## 3.6 生成正式标签文件

生成 `labels.csv`，字段建议包括：

```text
filename
original_path
cleaned_path
model_name
type
color
label_name
label_id
```

检查：

- [ ] 是否有缺失字段
- [ ] 是否有非法 type
- [ ] 是否有非法 color
- [ ] 是否有重复 filename
- [ ] label_id 是否正确
- [ ] cleaned_path 是否真实存在

输出：

- [ ] `data/labels/labels_v1.csv`

## 3.7 类别分布统计

统计：

- [ ] 每个 type 的样本数量
- [ ] 每个 color 的样本数量
- [ ] 每个 type-color 组合的样本数量
- [ ] 找出严重不足类别
- [ ] 决定是否补样本、删除、合并或降低展示优先级

输出：

- [ ] `docs/label_distribution_v1.md`
- [ ] `outputs/eval/label_distribution_v1.csv`

## 3.8 图像预处理

- [ ] 统一 resize / crop 到 256×256
- [ ] 检查车体是否明显偏移
- [ ] 检查图片是否损坏
- [ ] 检查输出尺寸是否统一
- [ ] 保留原始图到清洗图的映射

输出：

- [ ] `data/cleaned/images_256_v1/`
- [ ] `data/labels/image_mapping_v1.csv`

## 3.9 打包训练集
